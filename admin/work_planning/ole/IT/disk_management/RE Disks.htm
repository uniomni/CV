Hi Tony

 

Greg says that you are in the process of rationalising the way we are using disks on perlite.

Currently it is a grab-and-store system where things are very hard to manage and disks fill up due to large temporary files building up, for example.

 

If you want my five cents I think we need to do something along the following lines:

 

Reduce the number of people using particular disks: For example have a some allocated to RAMP and some to CIP and NRAP with possibility of some shared areas.

 

Secondly, we could have some sort of agreement that certain disks are reserved for certain purposes:

I see four main categories:

 

1: Personal work files, documents, software, admin, presentations – small stuff which is updated regularly and where we CANNOT afford a full disk. Not being able to save your documents is very annoying. Computer generated output should be banned from this area.

 

2: Output from computer simulations and some input data.. These files can get very large, but are often temporary. Should the disk fill, it is not as annoying as if disks under category 1 fills up.

 

3: Shared storage, mostly read-only stuff such as Wally’s ‘Risk Research Group’ area where admin files, minutes, dot points etc are stored for everyone to peruse.

 

4: Other stuff – e.g. Miriam’s backup of simulated data. Can be very large, but could ideally go to tape backups.

 

Ideally, of course one would have an automatic tape robot that would take old files off-line, but that is another story.

 

 

Happy to discus further
